pycharmm.dynamics

Functions to configure and run molecular dynamics

Corresponds to CHARMM command DYNAmics

See CHARMM documentation dynamc for more information

Functions

  • use_lang -- use Langevin dynamics
  • use_start -- use starting velocities described by iasvel
  • set_nprint -- change step freq to print and store energy data during runs
  • get_nprint -- return step freq to print and store energy data during runs
  • set_nstep -- change the number of steps to be taken in each dynamics run
  • get_nstep -- return the number of steps to be taken in each dynamics run
  • set_inbfrq -- change the freq of regenerating nonbonded list during runs
  • set_ihbfrq -- change the freq of regenerating the hydrogen bond list
  • set_timest -- set the time step for dynamics
  • set_akmast -- set the time step for dynamics in AKMA units
  • set_firstt -- set the initial temperature for dynamics runs
  • run -- run the dynamics

  69class OPTIONS(ctypes.Structure):
  70    """A ctypes struct to hold runtime dynamics settings
  71
  72    Attributes
  73    ----------
  74    ieqfrq : int
  75        The step frequency for assigning or scaling velocities to
  76        FINALT temperature during the equilibration stage of the
  77        dynamics run.
  78    ntrfrq : int
  79        The step frequency for stopping the rotation and translation
  80        of the molecule during dynamics. This operation is done
  81        automatically after any heating.
  82    ichecw : int
  83        The option for checking to see if the average temperature
  84        of the system lies within the allotted temperature window
  85        (between FINALT+TWINDH and FINALT+TWINDL) every
  86        IEQFRQ steps.
  87
  88        .eq. 0 - do not check,
  89                 i.e., assign or scale velocities.
  90
  91        .ne. 0 - check window,
  92                 i.e., assign or scale velocities only if average
  93                 temperature lies outside the window.
  94
  95    """
  96    _fields_ = [('ieqfrq', ctypes.c_int),
  97                ('ntrfrq', ctypes.c_int),
  98                ('ichecw', ctypes.c_int),
  99                ('tbath', ctypes.c_double),

 369def set_rngseeds(new_seeds):
 370    """Set seed for the random number generator
 371
 372    The seed for the random number generator used for
 373    assigning velocities. If not specified a value based on
 374    the system clock is used; this is the recommended mode, since
 375    it makes each run unique.
 376
 377    One integer, or as many as required by the random number
 378    generator, may be specified. See CHARMM documentation
 379    [random](<https://academiccharmm.org/documentation/version/c47b1/random>)
 380
 381    Parameters
 382    ----------
 383    new_seeds: list[int]
 384        new seeds for the random number generator
 385
 386    Returns
 387    -------
 388    bool
 389        success == True
 390    """
 391    nrand = get_nrand()
 392    typed_seeds = (ctypes.c_int * nrand)(*new_seeds)
 393    success = lib.charmm.dynamics_set_rngseeds(typed_seeds)
 394    success = bool(success)
 395    return success

 655def _configure(**kwargs):
 656    """Set dynamics parameters from a dictionary of names and values
 657
 658    Parameters
 659    ----------
 660    **kwargs:
 661        names and values from OPTIONS
 662
 663    Returns
 664    -------
 665    OPTIONS:
 666        a ctypes.Structure class for options that get set when
 667        dynamics runs, e.g. ieqfrq, ntrfrq and ichew
 668        this is to be passed into a call to the run function
 669    """
 670    valid_opts = dict(
 671        [('ieqfrq', 0),

 729def run(init_velocities=None, **kwargs):
 730    """Execute a dynamics run for the current system
 731
 732    Parameters
 733    ----------
 734    init_velocities: dict
 735        initial velocity for each atom; 'vx', 'vy', and 'vz' each list[float]
 736
 737    **kwargs: dict
 738        names and values from OPTIONS (see _configure function)
 739
 740    Returns
 741    -------
 742    pandas.core.frame.DataFrame
 743        a dataframe with index equal to step number and columns named for the
 744        traditional dynamics energy output entry names
 745    """
 746    natom = coor.get_natom()
 747    if init_velocities:
 748        init_vx = (ctypes.c_double * natom)(*(init_velocities['vx'][0:natom]))
 749        init_vy = (ctypes.c_double * natom)(*(init_velocities['vy'][0:natom]))
 750        init_vz = (ctypes.c_double * natom)(*(init_velocities['vz'][0:natom]))
 751
 752        out_vx = (ctypes.c_double * natom)()
 753        out_vy = (ctypes.c_double * natom)()
 754        out_vz = (ctypes.c_double * natom)()
 755    else:
 756        init_vx = None
 757        init_vy = None
 758        init_vz = None
 759
 760        out_vx = None
 761        out_vy = None
 762        out_vz = None
 763
 764    options = _configure(**kwargs)
 765    success = lib.charmm.dynamics_run(ctypes.byref(options),
 766                                      init_vx, init_vy, init_vz,
 767                                      out_vx, out_vy, out_vz)
 768
 769    if not init_velocities:
 770        out_vx = (ctypes.c_double * natom)()
 771        out_vy = (ctypes.c_double * natom)()
 772        out_vz = (ctypes.c_double * natom)()
 773        out_vw = (ctypes.c_double * natom)()
 774        natom = lib.charmm.coor_get_comparison(out_vx, out_vy, out_vz, out_vw)
 775
 776    out_velocities = pandas.DataFrame({
 777        'vx': [out_vx[i] for i in range(natom)],
 778        'vy': [out_vy[i] for i in range(natom)],
 779        'vz': [out_vz[i] for i in range(natom)],
 780        'vw': [out_vw[i] for i in range(natom)]})
 781
 782    return out_velocities

1205def get_msldata():
1206    """
1207    Get MSLD related data from the last step of a CHARMM dynamics run
1208
1209    Returns
1210    -------
1211    msldata : dict
1212        keys are 'steps', 'biases', 'blocks', 'nsubs' (if present) and 'thetas'.
1213

1215                 number of sites, temperature for lambda dynamics, functional form for
1216                 mapping between theta and lambda
1217
1218        'biases': settings related to viariable biases
1219
1220        'blocks': site index, lambda value of each substituent
1221
1222        'nsubs': number of substituents at each site
1223
1224        'thetas': theta of each substituent
1225

1250def _convert_form(form):
1251    new_form = 'NA'
1252    if form == 1.0:
1253        new_form = '2sin'
1254    elif form == 2.0:
1255        new_form = 'nsin'
1256    elif form == 3.0:
1257        new_form = '2exp'
1258    elif form == 4.0:
1259        new_form = 'nexp'
1260    elif form == 5.0:
1261        new_form = 'norm'
1262    elif form == 6.0:
1263        new_form = 'fixd'
1264
1265    return new_form

  • msldata (dict): keys are 'steps', 'biases', 'blocks', 'nsubs' (if present) and 'thetas'.

    'steps': general msld settings, eg., number of blocks, number of variable biases, number of sites, temperature for lambda dynamics, functional form for mapping between theta and lambda

    'biases': settings related to viariable biases

    'blocks': site index, lambda value of each substituent

    'nsubs': number of substituents at each site

    'thetas': theta of each substituent
